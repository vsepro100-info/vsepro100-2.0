# AGENTS.md

## Правила работы в этом репозитории

1. Истина по архитектуре и постановке живёт в документах репозитория.
2. Стартовая каноническая постановка находится в:
   `docs/CODEX_IMPLEMENTATION_MASTER.md`
3. До прямой команды нельзя начинать реализацию модулей, если задача была только на фиксацию документации.
4. Нельзя менять канонику полей, статусов, ролей, сущностей и связей по своей инициативе.
5. Нельзя смешивать:
   - WordPress role
   - business status
   - participation mode
   - entitlement
   - relationship link
6. Нельзя плодить альтернативные `meta_key`, статусы и поля с тем же смыслом.
7. Нельзя переносить legacy-хаос как новую канонику.
8. Каждый следующий шаг должен опираться на:
   - `duplication-core`
   - `duplication-access`
   - `duplication-webinars`
   - `duplication-agents`
   в утверждённом порядке реализации.
9. Любая реализация должна сначала сверяться с master-документом.
10. Если задача касается только документации — не начинать писать код модулей.
